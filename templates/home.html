Censorship Resistance

Secure Private Decentralization

Most organizations (e.g. Facebook, Google, Twitter, banks, governments) track, persist, and use our behavior for their gains.

Blockchain based applications and cryptocurrencies like Bitcoin are helping to wrestle some control from these centralized organizations although they are largely used on servers and distributed ledgers are still logically centralized creating an easy attack vector.

Smart phones, our primary means of global communication and collaboration, are weak in maintaining our anonymity and privacy - critical to ensuring individual freedom.

What's needed are decentralized applications that run purely on mobile phones without using servers of any kind and share only what the owner specifically allows while maintaining our privacy and ensuring transparency in our relationships.

1M5 is building a secure open-source decentralized peer-to-peer mobile application platform with end-to-end encryption and anonymity as a base layer for creating easy to use secure decentralized peer-to-peer smart mobile apps requiring no server connections that can be used around the world by any person looking to protect their communications and personal data from unjust intrusion, interception, monitoring, and censorship while isolating unethical actors from using the network.

Platform

Mobile Application Platform

1M5's platform provides mobile application infrastructure to build on. It's based on a Micro-Services Architecture (SOA) for service integration and a Staged Event-Driven Architecture (SEDA) for message routing. A number of common internal services are provided by the platform in addition to supporting pluggable services from service providers.

The following internal services are provided or currently in development:

Contracts

Think Ethereum turing-complete platform on a mobile using OpenDocuments for the contract language and a Rete rules engine for execution. When combined with Synaptic Celerity, it truly makes electronic automated contracts 'smart'.

Info Vault

The platform allows for the user to grant access control to their personal information on their mobile if desired. Access can be revoked when its no longer desired for them to have it. Applications no longer need to know everything about you. This will end widespread data breaches.

Key Ring

Maintaining encryption and signing keys secure.

Decentralized IDentifier

Pluggable, decentralized, people and/or automation-controlled universal ID, called DID or Decentralized IDentifier. Its primary components are:
Crypto Authentication (CA)
Reputation Based Access Control (RepBAC)
Circles of Influence

Prana - User Utility Token

Tokens used in the platform as a method to manage resources (network, cpu, storage) to support it.

Aten - Developer Utility Token

Tokens used in the platform as a method to determine transaction fee distribution paid in Prana.

Sensors

Mobile-to-Mobile (M2M), End-to-End Encrypted, Double Ratchet Key Management, Mesh Network Capable, Anonymous Network, with Bad Actor Isolation.

Content

Full-stack synched decentralized javascript web applications on android for decentralized content distribution including static resources.

Consensus

Unpermissioned blockchain on Android. Prevent double spend. Anyone can get on with their DID (public/private keys).

DEX

Trade tokens directly.

Social: Dgramz

Dgramz is a secure open-source decentralized peer-to-peer mobile messaging application running over 1M5 using Tor, I2P, I2P Bote, and Mesh networks with end-to-end encryption by default requiring no server connections protecting your communications and personal data from unjust interception, theft, and monitoring. No telephone numbers or email addresses are required. Only your public key is used and can be swapped using QR codes. Building up of reputations through Dgramz will provide you with other opportunities such as in 460R4 to approve transactions gaining tokens. Long-term roadmap includes features to replace popular centralized social media such as Instagram, Facebook, LinkedIn, and Twitter while providing social AI assistance when combined with Synaptic Celerity.

Governance: V4D

A decentralized governance application to assist with self-governance.

  • Voluntary: Who likes getting coerced or forced into something they don't wish to participate in?
  • Decentralized: Democracy works best with small groups as when it was first designed.
  • Delegative: We can not be experts in everything nor do we have the time to participate in every decision yet we have the right to attempt it.
  • Direct: No one person can represent another on every issue.
  • Democracy: Governance for the people by the people.

First module is a software development bounty application. Uses 1M5 to ensure voting can be done privately individually yet without losing transparency in the voting scores.
